# Radar v3 — spec de melhorias

Data: 2026-07-21. Insumo: `2026-07-21-auditoria-gaps.md` (IDs referenciados
abaixo). Sucede e complementa a spec de 2026-07-20, que permanece válida
onde não contradita.

## Princípio

A direção editorial de 20/07 (agenda com espinha de datas, Newsreader +
Archivo, hairlines, cor só como sinal semântico) está certa e fica. Esta
spec **executa a direção até o fim** — a espinha vira estrutura visível, os
cabeçalhos ganham voz editorial de verdade, os dados param de envergonhar o
design — e fecha os gaps funcionais que a auditoria mediu.

Dependência nova permitida: **lucide-react** (ícones tree-shaken, ~1KB por
ícone usado). Animação via **View Transitions API nativa** (zero
dependência, progressive enhancement) — framer-motion foi avaliado e
descartado: peso não se paga numa página única.

## 1. Design (escopos UI/TEC da auditoria)

### 1.1 Assinatura: a espinha de datas vira estrutura real

Hoje a "espinha" é só alinhamento de coluna. Passa a ser um **fio contínuo
com nós**, o elemento pelo qual a página é lembrada:

- Cada grupo temporal desenha uma hairline vertical (`--line`, 1px)
  entre a coluna de datas e o conteúdo, contínua do primeiro ao último item
  do grupo.
- Cada linha tem um **nó** no fio, na altura do título: disco de 5px.
  Cor do nó = urgência (`--critico` ≤3d, `--accent` 4–14d, `--line` no
  resto) — é o mesmo sinal da data, codificado em posição. Nada de novo a
  aprender, cor continua restrita à espinha.
- **Cor nunca sozinha** (CRIT-3): no nível crítico (≤3 dias) a data ganha
  também peso (`font-semibold`) — sinal redundante para daltonismo, como a
  pesquisa NN/g citada na spec de 20/07 já mandava.
- **"Sem prazo divulgado" não tem fio** — onde não há tempo, não há espinha.
  A ausência é semântica, não descuido. (O em-dash da coluna de data fica.)
- Implementação: pseudo-elementos no `<li>`/`<ul>` (border-left +
  `::before` do nó posicionado), zero markup extra.

### 1.2 Cabeçalhos de grupo editoriais (UI-5, TEC-2, UI-4)

`ESTA SEMANA` (uppercase + tracking + cinza) → **"Esta semana"** em
Newsreader itálico ~15px, `--ink`, caixa de frase, contador ao lado em
Archivo `tabular-nums` `--muted` (resolve o contraste 1,2:1 de UI-4).
O itálico do Newsreader já é pré-carregado (64KB hoje sem uso — TEC-2 se
resolve por uso, não por remoção).

### 1.3 Barra de controles (UI-1, UI-7, TEC-4/UI-6)

Reestruturada para eliminar a busca de largura zero:

```
┌──────────────────────────────────────────────────────────────┐
│ ⌕ Buscar edital…                              fonte: todas ▾ │  linha 1
│ minhas áreas   IA  Tecnologia  Saúde  Agro  …                │  linha 2
└──────────────────────────────────────────────────────────────┘
```

- Busca com `min-w` real e `flex-1` na própria linha (ícone `Search` do
  lucide, 14px, `--muted` — affordance, não decoração); select de fonte à
  direita.
- Chips de área na segunda linha, com `py-1 -my-1` (alvo de toque ≥24px,
  UI-7) e rótulo visível **"minhas áreas"** (TXT-5).
- A barra é **sticky** (top-0): numa página de 6.500px, filtrar sem rolar
  de volta é função, não enfeite. Fundo `--bg` com leve transparência +
  `backdrop-blur`, hairline inferior. `--surface` finalmente é usado — no
  hover de linha (1.4), não aqui.

### 1.4 Linha do edital (UI-6, FUN-4, FUN-5, DAD-4, a11y)

- **Estrutura de link corrigida**: hoje a linha inteira é um `<a>` com
  `<h3>` dentro e impede qualquer ação secundária. Passa a: `<li>` relativo,
  título é o `<a>` com pseudo-elemento esticado (stretched link — linha
  inteira continua clicável), ações secundárias ficam acima com z-index.
- **Hover de linha inteira**: `hover:bg-[var(--surface)]` +
  `transition-colors duration-150` (UI-6, UI-8). O sublinhado do título
  permanece.
- **Ação "agenda"** (FUN-4): botão-ícone `CalendarPlus` (15px) alinhado à
  direita da linha de metadados, `aria-label="Adicionar prazo à agenda"`,
  visível sempre; baixa `.ics` gerado client-side (lib pura, sem rede). Só
  em editais com prazo.
- **Badge "novo"** (FUN-5): prefixo `novo` em `--accent-forte`
  `font-medium` no início da linha de metadados, quando o edital apareceu
  depois da última visita do usuário (`radar:ultimaVisita` em
  localStorage; primeira visita não marca nada). Depende de DAD-3.
- **Público-alvo** (DAD-4): metadado discreto após as áreas, minúsculo:
  `· para empresas` / `· para ICTs`. Só FINEP tem o dado; ausência não
  renderiza nada.
- **"fluxo contínuo"** (DAD-1): editais assim rotulados mostram
  `fluxo contínuo` no lugar de `prazo no edital`.

### 1.5 Cabeçalho da página (FUN-3)

`Radar de Editais` + linha de stats ganha o terceiro segmento do mockup
original: `46 abertos · 5 fecham em 7 dias · atualizado hoje`.
"atualizado hoje/ontem/há N dias" deriva de `dados.atualizadoEm`
(calendário de São Paulo, helpers existentes); com ≥2 dias, cor `--critico`
— é o alarme de Action quebrada que hoje não existe.

### 1.6 "Esta semana" sempre presente (UI-3)

Sem filtro ativo, o grupo "Esta semana" renderiza mesmo vazio, com uma
linha de estado: `nenhum prazo fecha esta semana` (Archivo, `--muted`).
A pergunta nº 1 da página ("o que fecha logo?") sempre tem resposta
explícita. Com filtro ativo, grupos vazios continuam sumindo (o contexto é
outro: o usuário está recortando).

### 1.7 Movimento (UI-8)

- `transition-colors duration-150` em todos os interativos.
- Mudanças de filtro/busca embrulhadas em `document.startViewTransition`
  quando disponível e sem `prefers-reduced-motion` — crossfade suave da
  lista, zero dependência, degrada para troca instantânea.
- O bloco reduced-motion existente passa a proteger algo de verdade.

### 1.8 Identidade e metadados (UI-2, UI-10, TXT-7, CRIT-1, CRIT-2)

- `app/icon.svg`: marca do radar — dois arcos concêntricos + ponto, traço
  2px, em `--accent` ocre; `<style>` interno com media query para o tema
  escuro. Funciona a 16px.
- `app/opengraph-image.tsx`: `ImageResponse` estático no build — fundo
  `--bg`, "Radar de Editais" em Newsreader, linha de valor em Archivo,
  marca do radar. Fontes TTF subsetadas commitadas em `assets/`.
- `metadata`: `metadataBase` + blocos `openGraph` + `twitter`
  (summary_large_image) reusando title/description.
- `export const viewport`: `themeColor` claro/escuro (UI-10).
- **SEO técnico** (CRIT-1): `app/robots.ts` (allow all) + `app/sitemap.ts`
  (a única rota).
- **PWA mínimo** (CRIT-2): `app/manifest.ts` + `apple-icon` (PNG gerado da
  marca). Sem service worker — só o atalho de tela inicial.

### 1.9 Ajustes de token (UI-9, TEC-4)

- `--muted` claro: `#6a6e75` → `#62666d` (5,3:1, margem AA real).
- `--surface`: deixa de ser órfão (hover de linha, 1.4).
- Sem tokens novos além de uso; sem gradientes; raio segue contido.

### 1.10 Fora de escopo de design

Toggle manual de tema (auto por `prefers-color-scheme` basta), command
palette, virtualização de lista, ilustrações/empty states desenhados,
qualquer biblioteca de componentes.

## 2. Funcionalidades (escopo FUN)

### 2.1 Busca corrigida (FUN-1)

Em `filtrar()`:
- Termo normalizado com ≤3 caracteres casa por **palavra inteira**
  (fronteira Unicode), não substring.
- Termo exatamente `ia` também casa a flag `e.ia`.
- Termos longos seguem por substring (comportamento atual).
- Testes: "ia" → só editais de IA; "iot" palavra inteira; termo longo
  inalterado; acento-insensível preservado.

### 2.2 Preferência saneada (FUN-2)

`lerAreas()` recebe o conjunto de áreas válidas
(`Object.keys(AREAS) ∪ {'ia'}`), filtra o que não existe mais e persiste a
lista limpa de volta. Teste com área fantasma.

### 2.3 Filtros na URL (FUN-6)

Estado → query string (`?q=…&fonte=…&areas=a,b`) via
`history.replaceState` (sem hooks do Next, sem Suspense — mesmo padrão do
localStorage: lido no mount, escrito a cada mudança). URL tem precedência
sobre localStorage na chegada. Link compartilhado reproduz a visão.

### 2.4 Exportar prazo (FUN-4)

`lib/ics.ts`: `gerarIcs(edital): string` — VCALENDAR/VEVENT all-day na data
do prazo, UID estável (`id@radar-editais`), SUMMARY com título limpo,
URL + DESCRIPTION. Escapes de vírgula/ponto-e-vírgula/quebra testados.
Download via data URI (`download="prazo-<id>.ics"`).

### 2.5 Badge "novo" (FUN-5)

`radar:ultimaVisita` (ISO) em localStorage. `novo = coletadoEm >
ultimaVisita`. Atualizada no mount (após leitura). Primeira visita: nada
marcado. Depende do DAD-3 (coletadoEm = primeiro visto).

## 3. Dados & scraper (escopo DAD)

### 3.1 FINEP (DAD-1, DAD-4, DAD-7, DAD-10, DAD-12)

- Schema do item ganha `dataDePublicacao` (a API já ordena por ele).
- **Descarte de mortos**: sem prazo E `dataDePublicacao` (fallback
  `vigenciaInicio`) mais velho que **18 meses** → fora. Mata os itens de
  2017/2024 sem risco para editais recentes ainda sem prazo.
- **Fluxo contínuo**: título/descrição normalizados contendo "fluxo
  continuo" → `fluxoContinuo: true` no Edital (campo opcional novo no
  schema). Esses não são descartados por idade.
- **Descrição**: corte em ~600 chars em limite de palavra + "…" (função
  compartilhada com `resumir`/util). 
- `publicoAlvo: string[]` persistido (nomes como vêm da API).
- Dedupe de conteúdo: ver 3.4.

### 3.2 CNPq (DAD-5)

Visitar a página de cada chamada (~10 requisições/dia, `buscarTexto` com
retry existente): extrair primeiro bloco de texto do corpo como
`descricao` (mesmo teto de 600) e **reclassificar** com título+descrição.
Falha na página individual não derruba a fonte — segue sem descrição
(mesmo padrão do PDF da FAPEG). Fixture nova da página de detalhe.

### 3.3 FAPEG (DAD-9)

`prazoDoPdf` passa a devolver `{ prazo, texto }`; o coletor reclassifica o
edital com `titulo + origem + texto do PDF` (já baixado — custo zero).
Fixture de texto de PDF já existe; teste da reclassificação.

### 3.4 Merge (DAD-3, DAD-10, DAD-11)

- `mesclar()` preserva `coletadoEm` do registro anterior quando o `id` já
  existia → o campo passa a significar "primeiro visto" (destrava FUN-5 e
  conserta a ordenação de "Sem prazo").
- Dedupe secundário por (título normalizado + descrição normalizada),
  mantendo o de URL lexicograficamente menor (estável entre execuções).
- `situacao` normalizada no merge: prazo no passado → 'encerrado' (o JSON
  vira fonte de verdade; o front continua com o próprio cinto de
  segurança).

### 3.5 Classificador (DAD-2, DAD-8)

- `inovacao`/`inovador*` saem de `tecnologia` — inovação é o gênero do
  site, não uma área. Meta: tecnologia cai de 70% para algo que
  discrimina (~30-40%); medir e registrar no commit.
- `sustentab*` → termos com colocação ambiental (`sustentabilidade
  ambiental`, `desenvolvimento sustentavel`, mantém `meio ambiente`,
  `clima*`…) e exclusão do falso positivo "sustentabilidade econômica"
  (caso real vira teste).
- DAD-12 (IA diluída): aceito como trade-off do custo zero; registrado no
  README ("Aviso").

### 3.6 Datas (DAD-6)

`dataBrParaIso(fimDoDia)` e `fimDoDiaIso` gravam `T23:59:59-03:00` (fim do
dia de Brasília). `diasAte` não muda (já conta por calendário SP). Testes
dos dois helpers atualizados + caso de fronteira 21h-24h BRT.

## 4. Escrita (escopo TXT)

- **TXT-1**: `limparTitulo` v2 — aceita "Chamada|Chamamento",
  "Públic[ao]", órgão composto de vários tokens, texto entre o número e o
  separador, e o padrão CAPES "… - Edital nº N/AAAA" no fim. Os 3 títulos
  reais que falham hoje viram testes.
- **TXT-2**: descrição que começa repetindo o título tem o prefixo removido
  (comparação normalizada) antes de `resumir()`.
- **TXT-3**: "1 aberto" singular no header.
- **TXT-4**: `EFPC` entra em `SIGLAS`.
- **TXT-5**: rótulo visível "minhas áreas" (já em 1.3).
- **TXT-6**: linha da FAPEG no README menciona o PDF/cronograma; CNPq
  menciona página de detalhe (3.2).
- **TXT-7**: OG/twitter (já em 1.8).

## 6. Verificação

1. `npm test` verde (novos testes inclusos), `npm run lint`,
   `tsc --noEmit`, `npm run build`.
2. `npm run scrape` real: FINEP sem itens de 2017/2024 no "sem prazo";
   distribuição de áreas re-medida (tecnologia < 45%); CNPq com descrições;
   dedupe removendo a duplicata real; `coletadoEm` preservado na segunda
   execução consecutiva.
3. Screenshots desktop/mobile × claro/escuro comparados com os "antes"
   (guardados no scratchpad da sessão): espinha com nós, cabeçalhos
   editoriais, busca visível no desktop, barra sticky, hover, badge novo,
   ação de agenda.
4. Console do navegador sem erros; View Transition degradando sem API.
5. Lighthouse/contraste: contador de grupo e `--muted` ≥ 4,5:1.
